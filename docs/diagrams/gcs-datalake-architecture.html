GCS Data Lake — Cross-Project Access

Managed Folders + IAM: per-dataset read/write scoped to feeder and consumer projects

Managed Folders

  • Each dataset = one Managed Folder with its own IAM policy
  • Requires uniform bucket-level access (already enabled)
  • Nestable up to 15 levels deep
  • IAM applies to all objects with folder path as prefix

Feeder (RW) Access

  • Granted roles/storage.objectAdmin on the managed folder
  • Read, write, delete objects within that dataset
  • One feeder project owns each dataset
  • Cross-project via service account principal

Consumer (RO) Access

  • Granted roles/storage.objectViewer on the managed folder
  • Read-only — list and download objects only
  • Multiple consumer projects per dataset
  • Each consumer sees only its authorized datasets
